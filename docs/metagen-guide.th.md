# คู่มือ MetaGen (ภาษาไทย)

MetaGen เป็นเครื่องมือสร้าง **title / description / keywords / category** สำหรับภาพ stock photo
โดยใช้ AI อ่านภาพให้ แล้ว export ออกมาเป็นไฟล์ CSV ตามรูปแบบของแต่ละแพลตฟอร์ม
(Adobe, Shutterstock, Dreamstime ฯลฯ)

คู่มือนี้เขียนแบบ "ทำตามทีละขั้น" สำหรับคนที่ไม่ใช่โปรแกรมเมอร์

---

## 1. เปิดโปรแกรม

1. ดับเบิลคลิกไฟล์ **`metagen/MetaGen.html`**
2. โปรแกรมจะเปิดในเว็บเบราว์เซอร์ (Chrome / Edge แนะนำ)
3. **ไม่ต้องติดตั้งอะไรเพิ่ม และไม่ต้องต่อเซิร์ฟเวอร์** — ทุกอย่างทำงานในเบราว์เซอร์ของคุณล้วน ๆ

> 💡 ไฟล์เดียวจบ ถ้าจะย้ายเครื่องก็แค่ก๊อปไฟล์ `MetaGen.html` ไปด้วย

---

## 2. เลือก AI provider และใส่ API key

ที่แถบบนสุด (header) จะมีปุ่มเลือกผู้ให้บริการ AI 3 ตัว:

- **● Claude** (Anthropic) — แนะนำ คุณภาพดี
- **● OpenAI** (GPT)
- **● Gemini** (Google)

**ขั้นตอน:**

1. กดเลือก provider ที่ต้องการ
2. นำ **API key** ของ provider นั้นมาวางในช่อง "API Key"
   - Claude: ขึ้นต้นด้วย `sk-ant-api03-...`
   - OpenAI: ขึ้นต้นด้วย `sk-proj-...`
   - Gemini: ขึ้นต้นด้วย `AIzaSy...`
3. กดปุ่ม **Test →** เพื่อตรวจว่า key ใช้ได้ (จุดไฟจะเป็นสีเขียวถ้าผ่าน)
4. กดไอคอน 👁 เพื่อสลับซ่อน/แสดง key

### 🔒 หมายเหตุความปลอดภัย (สำคัญมาก)

- API key ของคุณ **เก็บไว้ในเบราว์เซอร์เครื่องคุณเท่านั้น** (localStorage)
- เวลา generate ภาพ คำขอจะ **ส่งตรงจากเบราว์เซอร์ไปยัง AI provider** ที่คุณเลือก
  — ไม่มี "เซิร์ฟเวอร์กลาง" ของใครมาคั่นกลาง ไม่มีใครเห็น key ของคุณ
- ใช้งานในเครื่องตัวเองเท่านั้น **อย่าแชร์ไฟล์ที่มี key ติดไปด้วย** และอย่า commit key ขึ้น GitHub

---

## 3. เลือกโมเดล AI

ในแท็บ **Generate** ด้านซ้าย หัวข้อ "🤖 AI Model" จะมี dropdown ให้เลือกรุ่นของ provider นั้น
แต่ละรุ่นมีความเร็ว/คุณภาพ/ราคาต่างกัน เช่น:

- **Claude:** Sonnet 4.6 (แนะนำ) · Haiku 4.5 (เร็ว ถูก) · Opus 4.8 (ดีสุด แพงสุด)
- **OpenAI:** GPT-5.4 (แนะนำ) · GPT-5.4 Mini · GPT-5.5 (ดีสุด) · GPT-5.4 Nano (ถูกสุด)
- **Gemini:** Gemini 3.5 Flash (แนะนำ) · Gemini 3.1 Pro · Gemini 2.5 Flash-Lite (ถูกสุด และทดสอบแล้วว่าเสถียร)

> ⚠️ การเข้าถึงรุ่นใหม่ (GPT-5 / Gemini 3) ขึ้นกับบัญชีของคุณ ถ้ารุ่นใหม่ใช้ไม่ได้
> ให้ลอง **gemini-2.5-flash-lite** ซึ่งเป็นตัวสำรองที่พิสูจน์แล้วว่าใช้ได้แน่นอน

---

## 4. ตั้งค่า sliders (ความยาว Title / Description / Keywords)

ในแท็บ **Generate** จะมีแถบเลื่อน (sliders) ให้กำหนดขอบเขตของผลลัพธ์:

| หัวข้อ | ความหมาย |
|---|---|
| 📝 **Title Length** | จำนวนตัวอักษรขั้นต่ำ/สูงสุดของชื่อภาพ |
| 🏷️ **Keywords** | จำนวน keyword ขั้นต่ำ/สูงสุด |
| 📄 **Description** | จำนวนตัวอักษรขั้นต่ำ/สูงสุดของคำบรรยาย |

ใต้หัวข้อ Keywords ยังมีสวิตช์เลือกชนิดคำ:

- **Single-Word** — คำเดี่ยว เช่น "sunset", "mountain"
- **Double-Word Phrases** — วลีสองคำ เช่น "golden sunset"
- **Multi-Word Phrases** — วลียาว เช่น "misty mountain valley"

นอกจากนี้ที่ "🌐 Output Language" เลือกภาษาของผลลัพธ์ได้ (English, Thai, ฯลฯ)

> 💡 ในแท็บ **Platforms** มีปุ่ม **"↳ Match sliders to this platform's limits"**
> กดแล้วระบบจะตั้ง sliders ให้ตรงกับขีดจำกัดของแพลตฟอร์มที่เลือกอัตโนมัติ

---

## 5. สีจัดอันดับ Keyword (Keyword Rank Colors)

หลัง generate เสร็จ keyword แต่ละคำจะมีสีบอก "อันดับความสำคัญ" (จุดสี) — AI เรียงคำที่เกี่ยวข้องที่สุดไว้ก่อน:

| สี | ความหมาย |
|---|---|
| 🟣 ม่วง (Violet) | Excellent — ดีเยี่ยม (กลุ่มต้น ๆ) |
| 🔵 น้ำเงิน (Blue) | Good — ดี |
| 🟢 เขียว (Green) | Normal — ปกติ |
| 🟡 เหลือง (Yellow) | Mediocre — พอใช้ |
| ⚪ เทา (Gray) | Bad — อ่อน |
| ⚪ ขาว (White) | Unknown — ไม่ทราบ |

ใช้สีเหล่านี้ดูคร่าว ๆ ว่า keyword ตัวไหนน่าจะมีพลัง SEO มากที่สุด

---

## 6. เลือกแพลตฟอร์มเป้าหมาย

ไปที่แท็บ **Platforms** ด้านซ้าย จะเห็นแพลตฟอร์มแบ่งเป็น 2 กลุ่ม:

- **✓ ทดสอบแล้ว (Tested & verified)** — 8 แพลตฟอร์มที่อัปโหลดจริงผ่านแล้ว:
  Adobe, Shutterstock, Freepik, Dreamstime, 123RF, Pond5, Vecteezy, General
- **🧪 ยังไม่ทดสอบ (Not yet tested)** — 6 แพลตฟอร์มที่ทำตามสเปคไว้แต่ยังไม่ได้ยืนยันด้วยการอัปโหลดจริง:
  iStock, Getty, Depositphotos, Alamy, Canva, Motionarray

กดที่ไอคอนแพลตฟอร์มเพื่อเปิด/ปิดการใช้งาน แพลตฟอร์มกลุ่ม "ยังไม่ทดสอบ" จะมีสีจาง ๆ และมีป้าย 🧪 กำกับ

เมื่อเลือกแพลตฟอร์มแล้ว กล่อง "ℹ️ Platform Specs" จะแสดงรายละเอียด (ขีดจำกัด title, จำนวน keyword, ฯลฯ)
ถ้าเลือกแพลตฟอร์มที่ยังไม่ทดสอบ จะมีคำเตือนให้ตรวจ CSV ก่อนอัปโหลด

---

## 7. กด Generate

1. ลากรูปมาวางในกล่อง drop zone กลางหน้าจอ หรือคลิกเพื่อเลือกไฟล์
   (รองรับ JPG, PNG, WEBP, GIF)
2. กดปุ่ม **✦ Generate** ที่มุมล่างซ้าย
3. รอ AI ประมวลผลทีละภาพ — แถบ progress จะบอกความคืบหน้า
4. ถ้าต้องการหยุดกลางคัน กด **✕ Cancel**
5. ปุ่ม **⟳ All** ใช้สร้างใหม่ทั้งหมดตั้งแต่ต้น

ผลลัพธ์จะแสดงเป็นตาราง: รูปย่อ, ชื่อไฟล์, title, description, category, keywords (พร้อมจำนวนตัวอักษร)

---

## 8. ติดตามค่าใช้จ่าย (Cost Tracker)

ที่ด้านล่างของแถบซ้ายมีกล่อง **💰 Session Cost** บอก:

- จำนวนภาพที่ประมวลผล
- ค่าใช้จ่ายเฉลี่ยต่อภาพ
- จำนวน token เข้า/ออก
- **ค่าใช้จ่ายรวม** (ประมาณการเป็น USD)

ตัวเลขนี้คำนวณจากราคาต่อ 1 ล้าน token ของแต่ละรุ่น (ดู `MODEL_PRICING` ในโค้ด)
ใช้เป็นค่าประมาณการ ไม่ใช่ใบเสร็จจริง

---

## 9. แก้ไขผลลัพธ์ทีละแถว

ทุกแถวในตารางมีปุ่ม ✏️ สำหรับแก้ title, description, และ keywords ด้วยตัวเอง
ก่อน export ตรวจให้แน่ใจว่าเนื้อหาถูกต้อง โดยเฉพาะ:

- title ไม่เกินขีดจำกัดของแพลตฟอร์ม (badge ตัวอักษรจะเป็นสีแดงถ้าเกิน)
- keyword ไม่ซ้ำและตรงกับภาพ

> 💡 คลิกที่รูปย่อเพื่อดูภาพขยาย (lightbox)

---

## 10. บันทึก / โหลด Session (Save / Load) — สำคัญ

งานในตารางจะ **หายเมื่อปิดหรือรีโหลดหน้า** ถ้ายังไม่ได้บันทึก ดังนั้น:

- กด **💾 Save** ที่แถบหัวตารางผลลัพธ์ เพื่อดาวน์โหลดไฟล์
  `MetaGen_Session_YYYY-MM-DD_HHmm.json`
  (ไฟล์นี้เก็บเฉพาะ **ข้อความ** title/description/keywords/category/ค่าใช้จ่าย — **ไม่เก็บรูป** จึงไฟล์เล็ก)
- คราวหน้าเปิด MetaGen แล้วกด **📂 Load** เลือกไฟล์ `.json` นั้น เพื่อเรียกผลลัพธ์กลับมา

**ข้อควรรู้เรื่องแถวที่โหลดกลับมา (restored):**

- แถวที่โหลดจากไฟล์จะไม่มีรูปจริง → ปุ่ม 🔄 regen ต่อแถวจะกดไม่ได้ และไม่มีรูปย่อ
- แต่ **export CSV ทำงานปกติ** เพราะใช้แค่ชื่อไฟล์
- ถ้ามีผลลัพธ์ค้างอยู่ ระบบจะถามยืนยันก่อนแทนที่ทั้งชุด
- ถ้าไฟล์ JSON เสีย จะขึ้นแจ้งเตือน error และไม่แตะข้อมูลเดิม

---

## 11. Export CSV

มี 2 ปุ่มที่มุมล่างซ้าย:

- **↓ CSV** — export เฉพาะแพลตฟอร์มที่เลือกใน "Export Format" (dropdown ในแท็บ Platforms)
- **↓ All** — export CSV แยกไฟล์สำหรับ **ทุกแพลตฟอร์มที่เปิดใช้งานอยู่** พร้อมกัน

ไฟล์จะถูกตั้งชื่อ `MetaGen_<ชื่อแพลตฟอร์ม>_YYYY-MM-DD.csv`
(สำหรับ Dreamstime ไฟล์จะชื่อ `MetaGen_Dreamstime_YYYY-MM-DD.csv` ซึ่งเป็นไฟล์ที่
**เครื่องมือ Dreamstime Metadata Tool** ต้องใช้ — ดู
[คู่มือ Dreamstime tool](./dreamstime-tool-guide.th.md))

> ⚠️ ถ้าใน export มีแพลตฟอร์มกลุ่ม "ยังไม่ทดสอบ" จะมีคำเตือนเด้งขึ้น
> ให้เปิดไฟล์ CSV ตรวจดูก่อนอัปโหลดจริง

แพลตฟอร์มบางตัวมีกล่องตั้งค่าเฉพาะที่จะโผล่ขึ้นมาเมื่อเลือก เช่น:

- **Freepik** — ตั้งค่า AI Generated / รุ่น AI ที่ใช้ / prompt
- **Vecteezy** — เลือก License (pro / free / editorial / blank)
- **Dreamstime** — เลือกชนิด batch (AI / ไม่ใช่ AI) และข้อความ AI disclosure
- **123RF** — รหัสประเทศ 2 ตัวอักษร
- **Pond5** — เจ้าของลิขสิทธิ์ / ราคา / editorial

## 12. วิธีเลื่อนแพลตฟอร์มจาก "ยังไม่ทดสอบ" → "ทดสอบแล้ว"

ถ้าคุณอัปโหลด CSV ของแพลตฟอร์มกลุ่ม 🧪 ขึ้นเว็บจริงสำเร็จแล้ว สามารถเลื่อนให้เป็น "ทดสอบแล้ว" ได้
(แก้แค่บรรทัดเดียว):

1. ทดสอบจริงก่อน — generate, export CSV ของแพลตฟอร์มนั้น, อัปโหลดบนเว็บจริง, ยืนยันว่า import ผ่าน
2. เปิดไฟล์ `metagen/MetaGen.html` ด้วยโปรแกรมแก้ข้อความ (เช่น Notepad)
3. ค้นหาคำว่า `const PLATS = [`
4. หาแพลตฟอร์มของคุณ แล้วเปลี่ยน `tier:'untested'` เป็น `tier:'verified'` เช่น:
   ```js
   {id:'Alamy', nm:'Alamy', ic:'Al', bg:'#1b4d3e', tier:'verified'},
   ```
5. บันทึกไฟล์ แล้วรีโหลด MetaGen ในเบราว์เซอร์ — แพลตฟอร์มจะย้ายไปกลุ่ม "ทดสอบแล้ว"
6. (แนะนำ) อัปเดตตารางใน [`docs/csv-formats.md`](./csv-formats.md) ให้ตรงกัน

> ไม่ต้องแก้ตัวเลขใน `PLAT_SPECS` — ตัวเลขขีดจำกัดเหมือนเดิม เปลี่ยนแค่ `tier` เท่านั้น
