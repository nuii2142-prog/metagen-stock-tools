# คู่มือ Dreamstime Metadata Tool (ภาษาไทย)

เครื่องมือนี้ทำหน้าที่ **ฝัง metadata (IPTC / XMP / EXIF) ลงในไฟล์ภาพ** สำหรับอัปโหลดขึ้น Dreamstime
โดยอ่านข้อมูลจากไฟล์ CSV ของ Dreamstime ที่ export มาจาก MetaGen

> 🛡️ **ไม่แตะไฟล์ต้นฉบับเด็ดขาด** — เครื่องมือจะ **คัดลอก** ภาพไปไว้ในโฟลเดอร์ใหม่ชื่อ
> "Dreamstime Ready ..." แล้วฝัง metadata ลงในไฟล์ที่คัดลอกเท่านั้น ไฟล์เดิมของคุณปลอดภัย

---

## ภาพรวม workflow ครบวงจร

```
MetaGen  ──►  export Dreamstime CSV  ──►  วาง CSV ในโฟลเดอร์รูป  ──►  รัน Dreamstime tool  ──►  อัปโหลด
```

1. **ใน MetaGen:** generate metadata แล้วกด export โดยเลือก format = **Dreamstime**
   จะได้ไฟล์ชื่อ `MetaGen_Dreamstime_YYYY-MM-DD.csv`
2. **ย้ายไฟล์ CSV** นั้นไปไว้ใน **โฟลเดอร์เดียวกับไฟล์รูป JPG** ของคุณ
3. **รันเครื่องมือ Dreamstime tool** (2 วิธี ดูด้านล่าง)
4. เครื่องมือจะสร้างโฟลเดอร์ **"Dreamstime Ready ..."** ใหม่ ที่มีรูปพร้อม metadata
5. **อัปโหลดรูปจากโฟลเดอร์ Dreamstime Ready** นั้นขึ้น Dreamstime

---

## 2 วิธีในการรัน

### วิธีที่ 1 — ดับเบิลคลิก `.hta` (มีหน้าจอ GUI) — แนะนำ

ดับเบิลคลิกไฟล์ **`Dreamstime Metadata Tool.hta`** จะเปิดหน้าต่างโปรแกรมที่มีปุ่มกดให้ใช้งานง่าย
(เหมาะกับคนที่ไม่อยากพิมพ์คำสั่ง)

> 📦 **ไฟล์เดียวจบ:** `.hta` ตัวนี้เป็นไฟล์ self-contained — ฝัง script และไอคอนไว้ในตัวแล้ว
> คุณก็อปแค่ไฟล์ `.hta` ไฟล์เดียวไปไว้ที่ไหนก็ได้แล้วทำงานได้เลย (ไม่ต้องมี `.ps1` หรือโฟลเดอร์
> `assets` ตามไปด้วย — มันจะคลายไฟล์ที่จำเป็นออกไปไว้ที่ `%APPDATA%\DreamstimeMetadataTool` ให้เอง)
>
> 📌 **อยากได้ไอคอนสวยบนเดสก์ท็อป:** เปิด tool แล้วกดปุ่ม **"📌 Desktop Shortcut"** มุมขวาบน →
> โปรแกรมจะสร้าง "ทางลัด" ที่มีไอคอน Dreamstime ไว้บนเดสก์ท็อปให้ ครั้งต่อไปดับเบิลคลิกทางลัดนั้นได้เลย
> (หมายเหตุ: ไฟล์ `.hta`/`.bat` เองจะโชว์ไอคอนทั่วไปของ Windows เปลี่ยนที่ตัวไฟล์ไม่ได้ —
> การสร้างทางลัดเป็นวิธีมาตรฐานที่ได้ไอคอนสวย)

### วิธีที่ 2 — ดับเบิลคลิก `.bat` (ไม่มี GUI)

ดับเบิลคลิกไฟล์ **`Dreamstime Embed - Double Click.bat`** จะเปิดหน้าต่างดำ (Command Prompt)
แล้วทำงานอัตโนมัติ:

- ใช้รูปทั้งหมดในโฟลเดอร์ที่ `.bat` วางอยู่
- หาไฟล์ `MetaGen_Dreamstime_*.csv` ตัวล่าสุดในโฟลเดอร์นั้นเอง
- โหมดเริ่มต้นคือ **AI / Adobe Firefly** (แก้ได้ที่บรรทัด `AI_MODE` และ `AI_MODEL` ในต้นไฟล์ `.bat`)

> 💡 วิธี `.bat` ง่ายและไว แต่ปรับค่าได้น้อยกว่า GUI ถ้าต้องการเลือกโฟลเดอร์/CSV/โหมดเอง ใช้วิธี `.hta`

## 3 ขั้นตอนในหน้าจอ GUI (`.hta`)

### ขั้นที่ 1 — เลือกโฟลเดอร์งาน (Choose your work folder)

- กด **Choose Folder** แล้วเลือกโฟลเดอร์ที่มีไฟล์ JPG ของคุณ
- กด **Open** เพื่อเปิดดูโฟลเดอร์นั้นใน File Explorer
- โปรแกรม **จำโฟลเดอร์ล่าสุด** ไว้ ครั้งถัดไปจะเปิดมาที่เดิม

### ขั้นที่ 2 — เลือก CSV และตั้งค่า AI

- โปรแกรมจะหาไฟล์ `MetaGen_Dreamstime_*.csv` ตัวล่าสุดในโฟลเดอร์ให้อัตโนมัติ
- กด **Choose CSV** เพื่อเลือกไฟล์เอง หรือ **Latest** เพื่อใช้ตัวใหม่สุด
- เลือกโหมด: **AI-generated images** หรือ **Not AI**
- เลือกรุ่น AI ที่ใช้สร้างภาพ (เช่น Adobe Firefly, Midjourney 5)
- ติ๊ก **Play sound when done** ถ้าต้องการเสียงแจ้งเตือนเมื่อเสร็จ

### ขั้นที่ 3 — Output และความคืบหน้า (Output and progress)

- โปรแกรมจะสร้างโฟลเดอร์ **"Dreamstime Ready <วันเวลา>"** ใหม่ทุกครั้งที่รัน
- กดปุ่มเขียว **Embed Metadata** เพื่อเริ่มทำงาน
- ระหว่างทำงานจะแสดงสถานะสด "Embedding image X of Y — ชื่อไฟล์"
- เมื่อเสร็จ โฟลเดอร์ output จะเปิดขึ้นอัตโนมัติ และปุ่ม **Open Output Folder** จะกดได้

## 🧪 Dry Run — ทดลองก่อนทำจริง

ปุ่ม **🧪 Dry Run** ใน GUI ใช้ "ตรวจสอบการจับคู่ระหว่าง CSV กับรูป" **โดยไม่เขียนอะไรลงไฟล์เลย**

ใช้เพื่อดูล่วงหน้าว่า:

- มีกี่แถวใน CSV ที่ **จับคู่กับรูปได้** (ok / dry-run)
- มีรูปไหน **หาไม่เจอ** (missing)
- มีชื่อซ้ำที่ **จับคู่กำกวม** (ambiguous)

แนะนำให้กด **Dry Run ก่อนทุกครั้ง** ถ้าผลดูถูกต้องแล้วค่อยกด **Embed Metadata** ทำจริง

---

## โหมด AI vs ไม่ใช่ AI (และข้อความ disclosure)

Dreamstime กำหนดให้ภาพที่สร้างด้วย AI ต้องระบุให้ชัด เครื่องมือนี้จัดการให้:

| โหมด | สิ่งที่เครื่องมือทำ |
|---|---|
| **AI-generated** | เติมข้อความ disclosure (ค่าเริ่มต้น `"AI-generated image."`) นำหน้า description, เพิ่ม keyword `"ai generated"`, และฝังแท็ก XMP `DigitalSourceType = trainedAlgorithmicMedia` ลงในไฟล์ |
| **Not AI** | ไม่เติม disclosure, ไม่เพิ่ม keyword AI, ไม่ฝังแท็ก AI — เหมาะกับภาพถ่ายจากกล้องหรือผลงานวาดเอง |

> ถ้า description มีคำว่า "AI-generated" อยู่แล้ว เครื่องมือจะไม่เติมซ้ำ

---

## 🛡️ ย้ำอีกครั้ง: ไม่แตะไฟล์ต้นฉบับ

- เครื่องมือ **ห้าม** ใช้โฟลเดอร์ output เป็นโฟลเดอร์เดียวกับโฟลเดอร์รูปต้นฉบับ (จะ error ถ้าตรงกัน)
- ทุกครั้งจะสร้างโฟลเดอร์ **"Dreamstime Ready <วันเวลา>"** ใหม่ — ถ้าชื่อซ้ำจะต่อท้าย (2), (3)
- รูปต้นฉบับของคุณไม่ถูกแก้ไขหรือลบ

## อ่านไฟล์รายงาน `dreamstime-embed-report.csv`

หลังรันจริง (ไม่ใช่ Dry Run) จะมีไฟล์ **`dreamstime-embed-report.csv`** อยู่ในโฟลเดอร์ output
เปิดด้วย Excel เพื่อตรวจผลแต่ละไฟล์ คอลัมน์ **Status** มีความหมายดังนี้:

| Status | ความหมาย | ต้องทำอะไรไหม |
|---|---|---|
| **ok** | ฝัง metadata สำเร็จและตรวจอ่านกลับได้ครบ | ✅ ใช้ได้เลย |
| **dry-run** | (เฉพาะตอน Dry Run) จับคู่รูปได้ แต่ยังไม่ได้เขียนจริง | ℹ️ แค่ทดลอง |
| **missing** | หารูปตามชื่อใน CSV ไม่เจอในโฟลเดอร์ | ⚠️ ตรวจชื่อไฟล์/วางรูปให้ครบ |
| **ambiguous** | มีรูปชื่อตรงกันหลายไฟล์ จับคู่ไม่ได้ | ⚠️ แก้ชื่อให้ไม่ซ้ำ |
| **invalid** | แถวใน CSV ขาด title / description / keywords | ⚠️ กลับไปแก้ใน MetaGen |
| **write-error** | ExifTool เขียน metadata ไม่สำเร็จ | ⚠️ ดูข้อความใน column Message |
| **verify-warning** | เขียนแล้ว แต่ตรวจอ่านกลับได้ไม่ครบทุกฟิลด์ | ⚠️ เปิดไฟล์ตรวจ metadata เอง |

แถวที่เป็น **missing / ambiguous / invalid / write-error** จะถูก **ข้าม** (ไม่มีไฟล์ใน output)
ส่วน **ok / verify-warning / dry-run** จะมีไฟล์ผลลัพธ์

## การแก้ปัญหา (Troubleshooting)

### ExifTool — ครั้งแรกต้องต่อเน็ต

- เครื่องมือใช้ **ExifTool** ในการฝัง metadata ถ้าเครื่องยังไม่มี ExifTool โปรแกรมจะ
  **ดาวน์โหลดตัวพกพา (portable) อัตโนมัติ** จาก exiftool.org มาเก็บในโฟลเดอร์ `tools\exiftool`
- **ครั้งแรกจึงต้องมีอินเทอร์เน็ต** ครั้งต่อ ๆ ไปใช้ตัวที่โหลดไว้แล้ว ไม่ต้องต่อเน็ต
- ถ้ามี ExifTool ติดตั้งไว้แล้ว (ใน PATH หรือใน Program Files) เครื่องมือจะใช้ตัวนั้นเลย

### ถ้า Windows บล็อก HTA

- ถ้าเปิด `.hta` แล้วขึ้นว่า "Windows blocked local app permissions" หรือทำงานไม่ได้
  ให้แน่ใจว่าคุณ **ดับเบิลคลิกเปิดไฟล์โดยตรง** ไม่ใช่ลากไปเปิดในเว็บเบราว์เซอร์
- HTA ต้องรันผ่าน `mshta` ของ Windows เท่านั้น

### Execution policy ของ PowerShell

- ไม่ต้องตั้งค่า PowerShell execution policy เอง — ทั้ง `.hta` และ `.bat`
  เรียก PowerShell ด้วย `-ExecutionPolicy Bypass` ให้แล้ว สคริปต์จึงรันได้ทันที

### หา `dreamstime-embed.ps1` ไม่เจอ (เฉพาะ `.bat`)

- ปกติ `.bat` หาสคริปต์ในโฟลเดอร์เดียวกับตัวมันเอง
- ถ้าย้าย `.bat` ไปไว้ที่อื่น ให้แก้บรรทัด `INSTALL_DIR` ในต้นไฟล์ `.bat` ให้ชี้ไปโฟลเดอร์ที่มี
  `dreamstime-embed.ps1` (ค่าเริ่มต้น: `%USERPROFILE%\Documents\Fable Metagen\dreamstime-tool`)
- ทางที่ง่ายที่สุดคือ **วาง `.bat`, `.hta`, และ `.ps1` ไว้ด้วยกันเสมอ**
